# 修正：UI 文案統一為「節拍 / 約定」（上一輪做反了）

## 背景
ROADMAP 命名語彙表：**節拍 = 紀念日（anniversaries）**、**約定 = 想一起做的事（wishlists）**。UI 文案應一律用「節拍」「約定」。上一輪 V8 誤把方向做反，現在 UI 顯示成「紀念日 / 願望」，其中「願望」更是自創詞。請全部改正。

## 硬性規則
- **只改使用者可見的 UI 文案字串**；程式代號 / 函式名 / 資料欄位（anniversaries、wishlists、buildWishlistSection、deleteAnniversary…）**一律不動**。
- 目標詞固定：紀念日 → **節拍**；想一起做的事 → **約定**；願望 → **約定**。不要再出現「紀念日」「想一起做的事」「願望」在任何使用者可見文字。

## before → after（逐項）
- `characterPage.js`：sectionEl('紀念日') → '節拍'；placeholder「紀念日名稱（例如：相遇紀念日）」→「節拍名稱（例如：相遇紀念日）」；「還沒有紀念日。」→「還沒有節拍。」；「未命名紀念日」→「未命名節拍」；confirm「確定要刪除紀念日「…」嗎？」→「…刪除節拍「…」…」；刪除角色 hint 內「…記憶、紀念日、想一起做的事與相處統計…」→「…記憶、節拍、約定與相處統計…」。
- `homeView.js`：空狀態「…聲痕、拾貝、紀念日與願望會集中在這裡。」→「…聲痕、拾貝、節拍與約定會集中在這裡。」；摘要卡 title '願望' → '約定'；openCharacterModal('願望', …) 的標題字串 → '約定'；若節拍卡標題被改成「紀念日」也改回「節拍」。
- `store.js`：'未命名紀念日' → '未命名節拍'（預設標題，使用者可見）。
- 其餘 grep：全專案 src 搜「紀念日」「想一起做的事」「願望」，凡是**使用者可見字串**一律改為節拍/約定。promptBuilder 對 AI 注入的 fallback（a.title || '紀念日'）改為 '節拍' 即可（保持一致）。

## 驗收
1. grep src 「願望」= 0；「想一起做的事」= 0。
2. grep src 使用者可見的「紀念日」= 0（資料欄位 anniversaries 與註解不算）。
3. 首頁摘要卡顯示「節拍」「約定」；點開 modal 內標題、placeholder、空狀態、刪除確認皆為節拍/約定。
4. 程式代號未改、node --check 通過、無 console error。
只改 UI 文案字串，回報上述驗收。
